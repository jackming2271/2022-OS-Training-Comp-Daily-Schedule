**经验总结🐝:**
1. Rust不同于C++, 遇到不懂的Rust概念时, 可以结合操作系统或者编译的知识去理解, 最好不要借鉴C++ 
2. 我写的Rust代码大概率首次是不会通过编译的😈, 可以先使用 cargo build --release 检查编译问题, 这样比make run速度会快很多
3. 代码需要高内聚, 这一点在lab2做得不好(在task模块做了很多mm模块的操作), 之后一定严格要求, 尽量在模块内部实现并以接口的方式提供给调用方
